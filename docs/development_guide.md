# Development Guide — Sigil & Script

## Prerequisites

- **Node.js**: `20.19.0` or higher (the project uses `.nvmrc` if present; `nvm use` otherwise pick 20+)
- **npm**: comes with Node.js
- **OpenSpec** (for spec-driven workflow): `npm install -g @fission-ai/openspec@latest`

---

## Project Setup

```bash
# 1. Clone the repository
git clone <repo-url>
cd sigil-and-script

# 2. Install dependencies
npm install

# 3. Start the dev server
npm run dev
```

The dev server starts at `http://localhost:5173` by default (Vite).

---

## Available Scripts

| Script | Description |

## Working with OpenSpec

OpenSpec provides the `/ff`, `/apply`, `/verify`, `/archive`, and `/commit` workflow commands.

### Typical Feature Flow

```bash
# Optional: enrich a vague idea into a spec-ready story
/enrich-us <story or idea text>

# Create OpenSpec artifacts for the feature
/ff <feature-name-or-ticket>

# Implement one task at a time
/apply <feature-name-or-ticket>

# Validate against the spec
/verify <feature-name-or-ticket>

# Archive the completed change
/archive <feature-name-or-ticket>

# Commit and (optionally) open PR
/commit
```

## Adding New Components

1. Create `src/components/<area>/ComponentName.tsx`
2. Define a `ComponentNameProps` interface at the top of the file
3. Export a named functional component
4. If it reads global state, use `useSigil()` or `useSigilSelector()`
5. If it introduces new state, add the action to `SigilAction` and handle it in `sigilReducer`
6. Add a co-located test file `ComponentName.test.tsx`

## Adding New Design Tokens

1. Add to `src/styles/tokens.css` under the appropriate group
2. Use the token in component CSS files via `var(--token-name)`
3. Update `docs/data-model.md` if the token represents a new semantic concept

## Adding New Types

1. Add to `src/types/sigil.types.ts`
2. Update `docs/data-model.md` to document the new type

## TypeScript Strictness

`tsconfig.app.json` enforces strict mode. Common requirements:
- No implicit `any` — every parameter and return type must be explicit
- No non-null assertions without a guard comment
- `import type { ... }` for type-only imports

Run `npm run build` to catch TypeScript errors before committing.

## Git Workflow

- **Base branch**: `master`
- **Feature branches**: `feature/<short-description>`
- **Commit style**: imperative, English, subject under 72 chars (see `ai-specs/skills/commit/SKILL.md`)
- **No force-push to `master`**
